Vibe Themer — demo refinement options

A refinement pass on the README demo, and the "pop as the paint front passes" idea you suggested. All three share the same refined motion; they differ only in the per-token pop.

Mock VS Code in HTML/CSS/JS · deterministic, frame-stepped · headless Chromium + ffmpeg · 820×513 · 15 fps · ~3 MB each. Nothing here changes the shipped GIF.

What's new in the refinement (in all three): the recolor is now a spatial paint wave that sweeps top-to-bottom through the code (tokens recolor in document order, not by category); a brief “Analyzing your vibe” think-beat before color starts; the chrome (bars, background) settles first, then the code paints in; a downward light wipe synced to the wave; an editor caret for life; and smoother easing throughout.

1 · Refinement only no pop

The refined wave with a soft glow as each token lands — but no scale/lift. Calmest, most “serious tool.”

2 · Refinement + pop recommended

Each token gently lifts toward you, scales a touch, and glows as the paint front reaches it. Lift-dominant, so adjacent tokens never collide. Lively but still tasteful.

3 · Refinement + bold pop stronger

Same idea, turned up — bigger lift, more scale and glow. More playful; reads clearly even at a glance.

0 · Currently shipped for reference

The version in PR #25 — category-by-category recolor, diagonal sweep, no per-token pop.
